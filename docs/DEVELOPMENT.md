## 開発方針

- 素直でシンプルなコード
  - 複雑だがパフォーマンスが良いコードより、シンプルで直感的なコードを優先する
  - 早すぎる最適化は避ける
- ドキュメントファースト
  - doc-comment で必ず説明を書く
  - example は必ず書く
- テストファースト
  - 実装の前にテストで仕様を明確にする
  - 必ず仕様を満たすことを保証する最小限のテストを書く
  - 副作用とロジックは分離することで、テスト容易性を高める
- ログファースト
  - 適切なログレベルでログを出力する
  - デバッグで必要な情報をログに含める
  - 構造化ログを使用する

## コードスタイル

- 未実装であったり、簡略化された部分には `TODO` コメントや `todo!()` マクロを使用して明示する

## Git コミット

- Git コミットメッセージは英語で書く
- Conventinal Commits に従う
